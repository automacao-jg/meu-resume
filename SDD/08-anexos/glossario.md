# Glossário

| Termo | Definição |
|---|---|
| **SDD** | Software Design Document — documento que descreve a arquitetura e design de um sistema |
| **MVP** | Minimum Viable Product — versão mínima do produto com funcionalidade core |
| **BaaS** | Backend as a Service — Supabase provê auth, banco e storage sem gerenciar servidor |
| **Currículo BR** | Currículo no formato brasileiro (com foto, dados pessoais, objetivo) |
| **Currículo CA** | Currículo no padrão canadense (sem foto, sem dados pessoais, foco em resultados) |
| **Cover letter** | Carta de apresentação enviada junto com o currículo para uma vaga específica |
| **RLS** | Row Level Security — mecanismo do PostgreSQL para controle de acesso a nível de linha |
| **JWT** | JSON Web Token — formato de token de autenticação stateless |
| **Crédito** | Unidade de consumo do sistema: 1 conversão = 1 crédito |
| **Serverless** | Arquitetura onde o código roda em funções efêmeras (sem servidor dedicado) |
| **docx** | Formato de arquivo Word (.docx) — saída principal do sistema |
| **LGPD** | Lei Geral de Proteção de Dados (Brasil) — regula o tratamento de dados pessoais |
| **OCR** | Optical Character Recognition — leitura de texto em imagens/scans |
| **Core Competencies** | Seção do currículo canadense com lista de competências-chave |
| **Professional Summary** | Parágrafo de apresentação no topo do currículo canadense (4 linhas máx.) |
